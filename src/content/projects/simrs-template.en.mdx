Every SIMRS project usually starts from scratch on the same things: module structure, routing, state management, data tables, user roles. This repetitive work eats time before any actual clinical feature is touched. This template grew out of building real hospital systems — distilled into a reusable foundation.

## What I built

A frontend-only SPA template with 19 modules that mirror real hospital workflows:

- **Services** — registration, ER, outpatient clinics, inpatient, operating room
- **Support** — laboratory, radiology
- **Pharmacy** — dispensary, inventory
- **Finance** — cashier, accounting, reports
- **Integration** — BPJS, plus master data modules

Complete with 32 design-system components, role-based RBAC, bilingual i18n, and 230 tests.

## Technical decisions

**Pluggable backend.** During demos, MSW (Mock Service Worker) serves consistent fake data — the system runs fully without a server. In production, the same layer points to a real REST API. The frontend doesn't need to know the difference.

**Type-safe down to routing.** TanStack Router with file-based routes, TanStack Query for server state, Zustand for UI state, React Hook Form + Zod for validation. Each layer chosen so errors are caught at compile time, not while a doctor is using the system.

**Built to be handed off.** An onboarding wizard, component catalog, and module documentation are included so another team can pick it up without having to ask.

## Outcome

The template runs with 230 tests and held Lighthouse targets (accessibility and layout stability). Released open source under MIT so it can serve as a starting point for other SIMRS projects.
